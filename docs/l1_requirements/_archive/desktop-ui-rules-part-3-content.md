# Desktop UI Rules Part 3 — P7-P12 (form / hierarchy / discoverable / interrupt / non-disturbance / aesthetic)

[desktop-ui-rules.md](./desktop-ui-rules.md) の続編。

## P9. 画面密度はタスクに合わせて決める

### 原則

情報量は「少なければ良い」「多ければ強い」ではなく、
その画面で行うタスクの性質に合わせて調整する。

### 理由

ミニマルさは万能ではない。
一覧性や比較が重要な画面では一定の密度が必要であり、
逆に単純な完了フローでは情報を絞った方が良い。

### 見落としサイン

- 余白は多いが判断材料が足りない
- 情報は多いが読み分けられない
- 一覧画面なのに比較しづらい
- 単純作業なのに情報が多すぎる

### 自己レビュー

- この画面のタスクは「読む」「比較する」「入力する」「選ぶ」のどれが中心か
- そのタスクに対して情報量は多すぎないか / 少なすぎないか
- 密度を上げた結果、読み順や操作性を失っていないか

## P10. 熟練者の効率も考える

### 原則

デスクトップアプリでは、発見可能性だけでなく反復効率も重要である。
頻繁に行う操作には、ショートカットや素早い導線があるとよい。

### 理由

同じ操作を何度も行うユーザーにとって、
毎回同じクリック経路をたどる UI は摩擦が大きい。
特に業務アプリや制作ツールでは効率が UX の一部になる。

### 見落としサイン

- 頻出操作なのに毎回深い階層を辿る
- ショートカットがなく、マウス移動が多い
- 熟練者向けの近道が一切ない

### 自己レビュー

- よく使う操作を素早く実行できるか
- ショートカットがある場合、見つけられる形で示されているか
- 効率化のために初心者の理解可能性を壊していないか

## P11. UI はコンテンツや作業の邪魔をしない

### 原則

UI フレームや装飾は、作業対象より目立ちすぎないようにする。
ユーザーが見たいのは、通常、ツールそのものではなく、
ツールを使って扱うコンテンツや結果である。

### 理由

サイドバー、ツールバー、装飾、影、色などが過剰になると、
本来注目すべき作業対象から視線を奪う。

### 見落としサイン

- コンテンツより装飾の方が目立つ
- 常時表示のパネルが多く、主領域が狭い
- 「便利そう」な要素が増え続けている

### 自己レビュー

- 一番見てほしい内容が一番目立っているか
- 補助 UI が主作業領域を圧迫していないか
- 非常時以外は隠せる補助要素を常時出していないか

## P12. 美しさは独自性より整合性から生まれる

### 原則

見た目を整えるときは、派手な独自表現を足すより、
既存のトークン、余白、整列、書体、コントラスト、部品の整合を高める。

### 理由

多くの UI で品質感を下げるのは「個性のなさ」よりも、
**微妙なズレの蓄積** である。
統一感は、局所的な派手さより強く品質を支える。

### 見落としサイン

- 余白や角丸が場所ごとに微妙に違う
- 同種要素なのにサイズや書式が揃っていない
- 単発の例外スタイルが増えている

### 自己レビュー

- 今回の調整は全体の整合性を高めているか
- 既存トークンや共通部品で表現できないか
- 好みで足した差分になっていないか
