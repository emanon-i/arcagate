# Arcagate UX デザインビジョン

作成: 2026-04-23 / 調査バッチ batch-31 (PH-143)

## 1. 背景と目標

Arcagate はデスクトップに散在した起動元を一箇所に集約する**個人用コントロールパネル**である。
機能面（ランチャー・ワークスペース・コマンドパレット）が充実してきた現時点で、
次のフェーズは **「触り心地」「気持ちよさ」「洗練されたわかりやすさ」** の追求である。

### 目標メトリクス（主観指標）

| 指標                 | 現状                   | 目標                                           |
| -------------------- | ---------------------- | ---------------------------------------------- |
| 開いたときの第一印象 | 機能的だが質素         | 「ちゃんとしたツール」という確信を即座に与える |
| 編集時の気持ちよさ   | 操作は通るが反応が薄い | D&D やリサイズが手に吸い付く感覚               |
| 長時間使用時の疲労度 | 問題なし               | 意図的に落ち着いた色・動きで疲労を出さない     |
| テーマ切替の楽しさ   | ダーク/ライトのみ      | プリセットで「雰囲気が変わる」体験             |

## 2. ゲーム UI/UX から採れる設計原則

ゲーム UI は「プレイヤーにとって快適な操作感」を極限まで追求した分野であり、
デスクトップツールに転用できる原則が多い。

### 2-1. Juice（反応の豊かさ）

> 「どんな操作も、何かが起きたことをプレイヤーに伝える」

- ボタンクリック → わずかなスケール変化
- リスト項目ホバー → 背景のなめらかなフェード
- ドロップ成功 → 軽い弾む動き

Arcagate への適用: **マイクロインタラクション**をすべての CTA に付与する。
`transition-all duration-150` + `hover:scale-[1.02]` レベルの小さな変化で十分。

### 2-2. フィードバックループ

ゲームでは「操作 → 状態変化 → 視覚/音声フィードバック → 次の操作」の
サイクルが必ず完結する。

デスクトップツールで壊れやすい箇所:

- ドラッグ中の「今どこに置けるか」の不明瞭さ
- 保存・削除後の「完了した」感の欠如

Arcagate への適用:

- ドラッグオーバー時のグロー/ハイライトを強化
- 破壊的操作（削除確認後）にトーストアニメーションを追加

### 2-3. モーションデザイン

ゲーム UI のモーション三原則:

1. **Purposeful（目的がある）**: 状態遷移を伝えるためだけに使う
2. **Responsive（即応）**: 入力から 100ms 以内に何かが動き始める
3. **Consistent（一貫）**: 同種の操作には同じ easing / duration

デスクトップツールへの転用では「速さ優先」。ゲームより短い duration で。

| 操作種別               | 推奨 duration | easing                       |
| ---------------------- | ------------- | ---------------------------- |
| ホバー・フォーカス     | 120〜150ms    | ease-out                     |
| パネル出現/消去        | 200〜250ms    | cubic-bezier(0.4, 0, 0.2, 1) |
| ダイアログ出現         | 200ms         | ease-out + scale 0.96→1      |
| ドラッグフィードバック | 80ms          | linear                       |
| ページ遷移             | 200ms         | ease-in-out                  |

### 2-4. サウンドデザイン

ゲームでは常時・毎操作に音が鳴る。デスクトップツールでは**これは NG**。

採用可否の原則:

- **毎アクション鳴るのは NG**（ホバー・クリック・入力など）
- **重要イベントのみ OK**（例: ランチャーで起動成功、インポート完了）
- **デフォルト OFF、設定でON に切替**を必須とする
- 音量は OS ボリュームに連動。専用スライダー不要。

### 2-5. 情報密度とコントラスト

ゲーム UI は高密度でも可読性を保つために:

- **階層的な視線誘導**: 最重要 → 中程度 → 補足の 3 段階
- **コントラスト比**: WCAG AA（4.5:1）以上を常に維持
- **視線が流れる方向**: 左→右、上→下を基本とし、重要な UI は左上に

Arcagate への適用:

- `--ag-text-primary` / `secondary` / `muted` / `faint` の 4 段階は維持
- アクセントカラーは「本当に注意を引きたい場所だけ」に制限

## 3. Arknights Endfield の視覚言語分析

### 3-1. 概観

- ジャンル: 3D ストラテジー RPG / インフラ建設
- UI テイスト: ハードサイファイ（SF 軍事系）+ 有機的な自然テクスチャの混在

### 3-2. 背景・テクスチャ表現

- **コンタライン（等高線）**: 地形の等高線を抽象化した線が UI 背景にオーバーレイされる
  - 薄い半透明ライン、opacity 5〜15%
  - 静止状態では動かず、パネル出現時に微かに流れる
- **ノイズテクスチャ**: 単色塗りのパネルにフィルムグレインや砂目状ノイズを重ねる
  - Gaussian ノイズ opacity 3〜8%
  - SVG filter の `feTurbulence` または CSS backdrop-filter で実現可能
- **境界線の光**: パネルの上端・左端に 1px の明るい線（ライトストローク）を入れる
  - ガラス的な質感の演出

### 3-3. カラーパレット

| 役割       | Endfield 系           | Arcagate 現行                    |
| ---------- | --------------------- | -------------------------------- |
| 背景ベース | 深い青灰 (#0d1117 系) | #090b10 (dark) / #f8f9fb (light) |
| サーフェス | 暗めの青灰レイヤー    | rgba(255,255,255,0.03〜0.05)     |
| アクセント | 青緑系シアン          | cyan (#22d3ee)                   |
| 警告/注意  | 琥珀系オレンジ        | #fbbf24 系                       |
| テキスト   | 冷たい白 (blue-tint)  | rgba(255,255,255,0.95)           |

→ 現行の `--ag-*` トークンは Endfield テイストとすでに近い。

### 3-4. タイポグラフィ

- 欧文: モノスペース + セリフなしサンセリフの混用
- 日本語: システムフォントで十分（カスタムフォントはパフォーマンス懸念あり）
- 特徴的なラベル: ALL CAPS + 0.1em 程度の letter-spacing（ `text-xs tracking-widest` ）

### 3-5. アニメーション特性

- パネルが出現する際: わずかに上からスライドイン (translateY -8px → 0) + フェード
- 選択状態: 背景に低opacity のアクセントカラー + 左ボーダーライン 2px
- ホバー状態: 背景 opacity を 1 段階上げる（surface-2 → surface-3 相当）

## 4. Ubuntu 系カスタムデスクトップの視覚言語分析

### 4-1. 概観

GNOME / KDE の高度なカスタマイズ、Picom / KWin コンポジタによる
視覚効果が特徴。r/unixporn で人気のスタイル。

### 4-2. 主要な視覚要素

| 要素                             | 内容                                              | Arcagate への適用可否                                       |
| -------------------------------- | ------------------------------------------------- | ----------------------------------------------------------- |
| **背景透過（Blur）**             | ウィンドウ背景に背景画像のブラー処理を適用        | ○ WebView2 では `backdrop-filter: blur()` で代替可能        |
| **壁紙背景**                     | アプリウィンドウ越しに壁紙が透けて見える          | △ Tauri のウィンドウ透過 + CSS 背景で実現可能（設定で切替） |
| **雨粒・パーティクルエフェクト** | 画面に雨粒や粒子が降るエフェクト                  | ✗ 常時は疲労感あり・Arcagate には不要                       |
| **Frosted Glass**                | ぼかし + 半透明でガラス風パネル                   | ○ `backdrop-filter: blur(12px)` + `bg-surface/80`           |
| **アクリル / ノイズ重ね**        | ブラー + ノイズテクスチャ = アクリル質感          | ○ CSS filter + SVG noise                                    |
| **ラウンドウィンドウ角**         | ウィンドウの角丸（Tauri の `decorations: false`） | ✓ 実装済み (`--ag-radius-window`)                           |
| **スクロールバー非表示**         | 細いオーバーレイスクロールバー                    | ○ CSS でカスタマイズ可                                      |

### 4-3. Arcagate として取り入れるべきでない要素

- **常時パーティクル / 雨エフェクト**: 疲労感が高く、ツールとして不適
- **不透明度が低すぎる**: 可読性が損なわれる（opacity < 0.7 の本文は NG）
- **フォント過剰カスタマイズ**: ローカルフォント依存は配布を複雑化する

## 5. 音・モーション・インタラクションフィードバックの採用方針

### 採用する

| 要素                         | 適用箇所                                    | 優先度           |
| ---------------------------- | ------------------------------------------- | ---------------- |
| マイクロインタラクション     | ボタン・カード・リスト項目のホバー/クリック | 高               |
| ダイアログ出現アニメーション | scale + fade                                | 高               |
| ドロップゾーンハイライト     | D&D 時のフィードバック                      | 高               |
| スムーズスクロール           | ライブラリ・ワークスペース                  | 中               |
| テーマ切替トランジション     | CSS transition on `:root`                   | 中               |
| 起動成功トースト改善         | 現行トーストに入出スライドを付与            | 中               |
| 背景ノイズテクスチャ         | ダークテーマ時の微細グレイン                | 低（オプション） |
| サウンド（起動成功・削除）   | 重要イベントのみ、デフォルト OFF            | 低（オプション） |

### 採用しない

- 常時アニメーション（コンタライン流れ等）: CPU/GPU 負荷あり
- 雨粒・パーティクル: ツール用途には疲労感が高い
- BGM・環境音: デスクトップツールに不要
- 毎クリックのサウンド: OS の効果音設定と競合・うるさい

## 6. Reduced Motion 配慮

- `prefers-reduced-motion: reduce` 対応を必須とする
- アニメーションは `transition: none` で無効化できるよう実装する
- `@media (prefers-reduced-motion: reduce)` を Tailwind の `motion-reduce:` クラスで制御

## 7. 実装優先順位ガイド

```
1. マイクロインタラクション（ホバー・クリック）— 設計バッチで標準化
2. ダイアログ/パネル出現アニメーション — 設計バッチで標準化
3. 背景ノイズ/テクスチャ — 背景レンダリング基盤バッチ
4. Frosted Glass / ブラー — 背景レンダリング基盤バッチ
5. サウンドフィードバック — 触感強化バッチ（オプション）
6. テーマプリセット (Endfield 調 / Ubuntu 透過調) — テーマバッチ
```
