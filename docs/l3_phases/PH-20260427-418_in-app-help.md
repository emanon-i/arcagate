# PH-418: in-app ヘルプ最小実装（Nielsen H10）

## 問題

batch-90 audit でも Codex review でも繰り返し指摘:

- **Nielsen H10 違反**: ヘルプ・ドキュメンテーションの in-app アクセス手段なし
- onboarding hint が一切ない（Library を初めて開いた人が次に何をすべきか不明）
- ホットキー一覧が見える場所なし（Raycast / Alfred は `?` キーで cheat sheet 表示が標準）
- 各画面の「これ何？」を即解決する手段なし

「他人が使って配布できる」品質バーから見て、これは critical（severity 3-4）。

## 改修

### 1. グローバルヘルプパネル（`?` キー or `Ctrl+?`）

`src/lib/components/help/HelpPanel.svelte`（新設）:

- どこでも `?` キーで開く（input フォーカス中以外）
- 表示内容:
  - 主要ホットキー一覧（Palette / Settings / Help 等）
  - 現在画面の操作リファレンス（Library なら「D&D で追加」「ダブルクリックで起動」等）
  - 公式リンク（README on GitHub）
- Esc / `?` で閉じる

### 2. 画面別 onboarding empty state

各主要画面の空状態:

- **Library 空**: `EmptyState` に「アイコンを D&D してアイテムを追加 / Settings > 取り込みフォルダで自動取り込み設定」 + 「サンプル追加」ボタン
- **Workspace 空**: 「編集モードで「ウィジェット追加」から始める」 + 「サンプルワークスペース追加」ボタン
- **テーマ未選択**: 既存表示 OK

`EmptyState.svelte` (batch-86 PH-388 で新設) を拡張、`actions` slot を追加。

### 3. ツールチップ強化（hover hint）

主要ボタン（Settings 歯車 / Help `?` / 編集モード切替 等）に `title` 属性 + 内容説明 + ホットキー併記:

- 例: `title="設定 (Ctrl+,)"`
- 例: `title="ヘルプ (?)"`

### 4. 初回起動 onboarding tour（最小）

`src/lib/components/setup/OnboardingTour.svelte`（新設、最小機能）:

- 初回 setup 完了直後に 1 回だけ表示
- 3 ステップ:
  1. 「`Ctrl+Shift+Space` でパレットを開けます」
  2. 「`?` キーでいつでもヘルプ」
  3. 「Settings から見た目を変更可能」
- スキップボタン + 「もう表示しない」設定

### 5. ヘルプコンテンツのデータソース

`src/lib/help-content.ts` を新設、画面別 / 機能別ヘルプを TS データとして集約。i18n 拡張ポイントを残す（現状日本語固定）。

## 解決理屈

- Nielsen H10 = 配布水準の絶対要件
- Raycast / Alfred の `?` cheat sheet パターンを採用 → ユーザ慣れ済みパターン
- onboarding は 1 回限り、慣れたユーザの邪魔にならない
- Codex Q5 の「in-app ヘルプ最小実装」直接対応

## メリット

- 「何ができるか分からない」状態を解消
- 配布後の README 読まないユーザでも操作可能
- ホットキー学習曲線が緩やかになる（discoverable）

## デメリット

- onboarding tour の maintenance（変更時に再表示制御）
- 翻訳負荷（i18n しないと配布ターゲットが限定）
