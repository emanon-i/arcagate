# PH-20260226-003: パワーユーザー拡張（M2相当）

## 目的

CLI・ワークスペース・MCP連携により、単なるランチャーから「コントロールパネル」へ進化させる。パワーユーザー向けの操作強化と、AIからの操作インターフェースを確立する。

## Phase 1 からの引き継ぎ事項

本フェーズ着手時に以下を対応すること:

| 項目 | 備考 |
|---|---|
| E2E テスト導入（WebdriverIO + tauri-driver） | L2 §1.4 で定義済み。Phase 1 では GUI E2E を行わず Service/Repository 層テスト + コンポーネントテストで代替した |
| `tauri::test::mock_builder` の再評価 | Windows バグ ([tauri#14723](https://github.com/tauri-apps/tauri/issues/14723)) の解消状況を確認し、採用可否を判断する |
| 視覚回帰テスト（screenshot diff）の検討 | UI デザインが安定していれば導入を検討する |

## 機能詳細

### F-20260226-012: コマンドパレット拡張（CLI・内蔵コマンド）

CLIからのアイテム操作、内蔵コマンド、カテゴリプレフィックスによる名前空間絞り込みを実装する。

**技術要素**:
- CLI バイナリ（Tauri本体とは別プロセス）。named pipeで稼働中のTauriアプリと通信し、同一Service Layerを経由する。Tauriアプリ未起動時はDB直接アクセスにフォールバック
- 内蔵コマンド: クリップボード履歴・スニペット・電卓
- `categories.prefix` フィールド（L2 §4: M2a用）
- CommandProvider trait の活用（PH-001で定義済み）

**受け入れ条件**:
- [ ] `arcagate run <name>` でCLIからアイテムを起動できる
- [ ] `arcagate list`, `arcagate search <query>` でアイテムを検索・一覧表示できる
- [ ] コマンドパレットでクリップボード履歴を検索・貼り付けできる
- [ ] コマンドパレットで電卓（簡易数式評価）が使える
- [ ] カテゴリプレフィックス（例: `gm:` でGamesカテゴリに絞り込み）が動作する

### F-20260226-013: ファイルシステム監視・パス自動追跡

登録済みアイテムのパス変更を自動追跡し、リンク切れを防止する。

**技術要素**:
- `watcher/` の FileWatcher trait 実装（PH-001で定義済みのtraitを具体化）
- notify crate（ファイルシステム監視）
- 登録パスの定期チェックまたはリアルタイム監視
- パス変更検知時のアイテム自動更新

**受け入れ条件**:
- [ ] 登録済みアイテムのexeパスが移動・リネームされた場合、同一ドライブ内で自動追跡できる
- [ ] パスが見つからないアイテムに警告マークが表示される
- [ ] ユーザーに手動でのパス修正を促す通知が出る
- [ ] 監視対象のフォルダを設定画面から指定できる

### F-20260226-014: ワークスペース

ユーザーが任意のページを作成し、ウィジェットを自由配置できるカスタマイズ可能なホーム画面。

**技術要素**:
- SvelteKit ファイルベースルーティング（ワークスペースページ）
- 内蔵ウィジェット: よく使うもの / 最近使ったもの / プロジェクト一覧 / 監視フォルダ
- テーマシステム: ダーク / ライト / カスタムカラー（shadcn-svelte CSS変数ベース）
- ウィジェット配置のドラッグ&ドロップ
- Git ステータス表示（プロジェクト一覧ウィジェット）
- テーマ インポート/エクスポート
- M2追加マイグレーション（ワークスペース・ウィジェット設定テーブル）

**受け入れ条件**:
- [ ] 複数のワークスペースページを作成・削除・名前変更できる
- [ ] 「よく使うもの」「最近使ったもの」「プロジェクト一覧」「監視フォルダ」の内蔵ウィジェットを配置できる
- [ ] ウィジェットをドラッグ&ドロップで自由に配置・リサイズできる
- [ ] テーマ（ダーク・ライト・カスタムカラー）を切り替えられ、シェイプ設定を変更できる
- [ ] プロジェクト一覧ウィジェットでGitステータス（ブランチ名・未コミット変更の有無）が表示される

### F-20260226-015: MCP連携

Arcagate自身がMCPサーバーを内蔵し、Claude Codeからアイテムの登録・起動・検索を自然言語で操作可能にする。

**技術要素**:
- MCP Server（JSON-RPC over stdio）
- Service Layer 経由（L2 §2.1: UI/CLI/MCP同一Service Layer）
- Tauri v2 パーミッションモデル（capabilities/scopes）
- MCPツール: `arcagate_search`, `arcagate_launch`, `arcagate_create`, `arcagate_list`
- セットアップ自動化（Claude Code設定ファイルへの自動登録）

**受け入れ条件**:
- [ ] Claude Codeから「〇〇を起動して」でアイテムを起動できる
- [ ] Claude Codeから「〇〇を登録して」でアイテムを新規登録できる
- [ ] Claude Codeからアイテムの検索・一覧表示ができる
- [ ] MCPサーバーのセットアップをClaude Codeへの指示だけで完結できる（claude_desktop_config.json等への自動登録）
- [ ] MCPツールごとにread/write権限がallowlistで定義されている（デフォルトread-only）
- [ ] 未許可のwrite操作はエラーを返す
- [ ] allowlistはconfig画面から設定変更できる

## Exit Criteria

- CLIからアイテムの検索・起動ができる
- ワークスペースに2ページ以上を作成し、内蔵ウィジェットを配置して日常運用できる
- Claude CodeからMCP経由でアイテムの登録・起動・検索を操作できる
- ファイル監視がパス変更を検知し、ユーザーに通知できる
