## 背景・目的

`pnpm verify` には vitest（ユニットテスト）が含まれるが、ci.yml には追加されていない。
e2e.yml は既に存在し main ブランチへの PR で E2E が動く。
ユニットテスト（`pnpm test`）が CI でスキップされると、回帰が気づかれないリスクがある。

## 制約

- `.github/workflows/ci.yml` のみ変更
- 既存ステップの順序を変えない
- Tauri build の前に挿入（ビルドが通る前提でテストも通るべき）

## 実装仕様

`Tauri build` ステップの直前に挿入:

```yaml
- name: Vitest (unit tests)
  run: pnpm test
```

## 受け入れ条件

- ci.yml に `pnpm test` ステップが含まれること
- `pnpm test` が unit テストを実行すること（package.json: "test": "vitest run"）
