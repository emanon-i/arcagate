# PH-20260226-004: OS統合・AI連携深化（M3/M4相当）

## 目的

OSレベルのエクスプローラー統合と、AIファイル管理により最終形態に到達する。右クリックメニューからのコマンド実行、大規模ファイルインデックス、セマンティック検索を実現する。

| 機能ID         | 機能名                           | 対応REQ          |
| -------------- | -------------------------------- | ---------------- |
| F-20260226-016 | コンテキストメニュー統合         | REQ-20260226-012 |
| F-20260226-017 | ファイルマネージャー・AI連携深化 | REQ-20260226-013 |

## 機能詳細

### F-20260226-016: コンテキストメニュー統合

エクスプローラーの右クリックメニューにArcagateの登録コマンドを統合する。2段階で実装。

**技術要素**:

- 第1段階: Windows「送る」（SendTo）フォルダへのショートカット登録
- 第2段階: Tauri v2 ネイティブコンテキストメニューAPI / Windows Shell Extension
- **第2段階の技術リスク**: Windows Shell ExtensionはCOM DLL実装が必要。Explorerプロセス内でロードされるため、クラッシュ時のインパクトが大きい。Rust COM crate (`windows-rs`) の成熟度を評価した上で着手判断する。工数は第1段階の3〜5倍を見込む
- 複数ファイル選択時の引数受け渡し
- コマンドライン長制限対策（L0 §12: 大量選択時は一時ファイル `paths.txt` 経由）

**受け入れ条件**:

- [ ] エクスプローラーの「送る」メニューからArcagate登録コマンドにファイルを渡せる（第1段階）
- [ ] エクスプローラー右クリックメニューにArcagateのサブメニューが表示され、登録済みコマンドを選択・実行できる（第2段階）
- [ ] 複数ファイルを選択して一括で渡すことができる
- [ ] 大量ファイル選択時（コマンドライン長制限超過時）は一時ファイル経由でパスを受け渡す

### F-20260226-017: ファイルマネージャー・AI連携深化

タブ付きファイルブラウザ、インメモリインデックスによる高速検索、MCPを通じたAIファイル操作を実装する。

**技術要素**:

- タブ付きファイルブラウザ UI（セッション記憶・ブックマーク）
- インメモリファイルインデックス（10万件規模ファジー検索）
- `watcher/` の FileWatcher trait 実装（PH-003で実装済みを拡張）
- ファイルシステムインデックスのMCP公開（MCPツール追加）
- セマンティック検索（「先週よく使ってたやつ」等）
- M4追加マイグレーション（ファイルインデックステーブル）

**受け入れ条件**:

- [ ] タブ付きファイルブラウザで複数フォルダを同時に開き、セッション終了後も復元される
- [ ] インメモリインデックスによるファジー検索が10万件規模で1秒以内に応答する
- [ ] ファイルシステムインデックスがMCP経由でAIに公開され、Claude Codeからファイル操作（タグ付け・フォルダ作成）ができる
- [ ] セマンティック検索（使用頻度・時期ベース）で関連ファイルを検索できる

## Exit Criteria

- エクスプローラー右クリックメニューからArcagate登録コマンドが動作する
- 10万件規模のファジー検索が1秒以内に応答する
- Claude CodeからMCP経由でファイル操作（タグ付け・フォルダ作成）ができる
- タブ付きファイルブラウザが日常のファイル操作に使える状態である

  - 特に F-20260226-013（ファイルシステム監視）と F-20260226-015（MCP連携）を前提とする
