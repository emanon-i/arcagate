# PH-003-I: クイック修正（サイドバー・ナビ + Library パネル）

## 概要

ユーザーフィードバックのうち、影響範囲が限定的で即着手可能な5件を一括修正する。
サイドバーのアイコンオンリー化、不要テキスト削除、Library 詳細パネルの操作性改善、センシティブ制御の文言修正、パスワード機能の削除を行う。

---

## I-1: LibrarySidebar をアイコンのみに変更

### 背景

現在の `LibrarySidebar` はアイコン＋テキストラベル＋件数メタを表示しており、横幅 200-250px を占有している。フィードバックでは「サイドバーはアイコンボタンのみ、左端に」とされている。

### 受け入れ条件

- [ ] `LibrarySidebar` がアイコンボタンのみを表示する（テキストラベル・件数メタなし）
- [ ] サイドバーの横幅が 48px 以下になる
- [ ] 各アイコンに `title` 属性（tooltip）でカテゴリ名を表示する
- [ ] アクティブ状態のアイコンが視覚的に区別できる
- [ ] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                    | 変更内容                                          |
| ----------------------------------------------------------- | ------------------------------------------------- |
| `src/lib/components/arcagate/library/LibrarySidebar.svelte` | テキストラベル・メタ削除、アイコンのみ表示に変更  |
| `src/lib/components/arcagate/common/SidebarRow.svelte`      | アイコンのみモード対応（label/meta を省略可能に） |
| `src/lib/components/arcagate/library/LibraryLayout.svelte`  | サイドバー幅のレイアウト調整                      |

---

## I-2: "Hidden Off" テキスト削除

### 背景

`+page.svelte` の AppHeader 右スロットに `TitleAction` として `label="Hidden off"` が表示されている。フィードバックでは「"Hidden Off" 文字不要」とされており、アイコンのみとすべき。

### 受け入れ条件

- [ ] AppHeader の "Hidden Off" テキストが非表示になる
- [ ] アイコンのみでトグル状態（表示/非表示）が視覚的に判別できる
- [ ] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                  | 変更内容                                      |
| ------------------------- | --------------------------------------------- |
| `src/routes/+page.svelte` | TitleAction の label 削除、アイコンのみに変更 |

---

## I-3: 空白クリックで詳細パネルを閉じる

### 背景

現在の `LibraryDetailPanel` は X ボタンでのみ閉じることができる。フィードバックでは「空白クリックで詳細パネルを閉じる」ことが求められている。

### 受け入れ条件

- [ ] LibraryMainArea のアイテム外（空白領域）をクリックすると詳細パネルが閉じる
- [ ] アイテムカードのクリックは従来通り詳細パネルを開く（イベント伝播を正しく制御）
- [ ] X ボタンによる閉じる機能は引き続き動作する
- [ ] E2E テストが通過する
- [ ] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                     | 変更内容                                                     |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| `src/lib/components/arcagate/library/LibraryLayout.svelte`   | 空白クリック時に `selectedItemId = null` を設定              |
| `src/lib/components/arcagate/library/LibraryMainArea.svelte` | 空白領域のクリックハンドラ追加（カードクリックとの伝播制御） |

## I-4: "センシティブ制御" 文言変更

### 背景

`SensitiveControl.svelte` に「非表示アイテムを隠す / 表示」の文言があるが、フィードバックでは「"センシティブ制御" 文言やめて」とされている。「可視化/不可視化」に変更する。

### 受け入れ条件

- [ ] `SensitiveControl` の表示テキストが「可視化」/「不可視化」に変更される
- [ ] コンポーネント名 `SensitiveControl` はそのまま維持（ファイル名リネームは不要）
- [ ] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                      | 変更内容                                    |
| ------------------------------------------------------------- | ------------------------------------------- |
| `src/lib/components/arcagate/library/SensitiveControl.svelte` | 表示テキストを「可視化」/「不可視化」に変更 |

---

## I-5: パスワード機能削除

### 背景

フィードバックで「パスワード機能不要」とされている。`HiddenPasswordForm` 関連コンポーネントが存在する場合は削除する。調査の結果、`HiddenPasswordForm.svelte` は存在しないが、パスワード関連の残存コードがないか確認し、あれば削除する。

### 受け入れ条件

- [ ] パスワード関連の UI コンポーネント・ロジックが存在しないことを確認
- [ ] `SensitiveControl` のトグルがパスワードなしで直接動作することを確認
- [ ] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル | 変更内容                             |
| -------- | ------------------------------------ |
| 調査対象 | パスワード関連コードの残存確認・削除 |
