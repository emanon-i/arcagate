## 背景/目的

Library の `/` キーフォーカス・クリアボタン・フィルタリングに対するテストが存在しなかった。
実機フィードバック PH-034（フォーカス）・PH-036（クリアボタン）の回帰を防衛するため追加する。

## 制約

- `tests/e2e/library-search.spec.ts` を新規作成
- `resizeWindow(page, 1280, 800)` でレイアウト安定化
- `waitForAppReady` でデータロード待機

## 手法

| # | テスト                                                 |
| - | ------------------------------------------------------ |
| 1 | `/` キーで検索 input にフォーカス                      |
| 2 | 入力フィールド内で `/` を押しても「/」が入力されるだけ |
| 3 | キーワード入力でアイテムが絞り込まれる                 |
| 4 | クリアボタンで入力リセット＋アイテム再表示             |
| 5 | クリア後に検索 input にフォーカスが戻る                |
