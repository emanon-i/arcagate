## 背景/目的

PH-20260422-040 でウィジェット削除フローを「即削除」→「確認ダイアログ経由」に変更したため、
既存の E2E テストが削除ボタンクリック直後に削除を期待していて失敗するようになった。
テストを新しい UX フローに合わせて修正し、キャンセルパスもカバーする。

## 制約

- E2E ファイルのみ変更（プロダクションコード不変）
- 削除確認ダイアログの aria/text ラベルはプロダクション実装に合わせる

## 手法

| # | 変更内容                                                            |
| - | ------------------------------------------------------------------- |
| 1 | 既存テスト「ゴミ箱ボタン即削除」→「確認ダイアログ経由で削除」に更新 |
| 2 | 新テスト「キャンセルでウィジェットが残ること」追加                  |

## 受け入れ条件

- [x] 確認ダイアログが表示されること (`role="dialog"`)
- [x] 「ウィジェットを削除しますか？」テキストが表示されること
- [x] 「削除」ボタンでウィジェット数が -1
- [x] 「キャンセル」でダイアログが閉じ、ウィジェット数が変わらないこと
- [x] `pnpm verify` 全通過
