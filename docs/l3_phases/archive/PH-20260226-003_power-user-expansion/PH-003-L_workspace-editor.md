# PH-003-L: ワークスペース編集 UI

## 概要

ワークスペースのウィジェット配置を直感的に編集できる UI を実装する。
編集用左サイドバー、D&D によるウィジェット追加、編集中のグリッド線表示、
ウィジェットのリサイズ/移動 UI を新規に構築する。PH-003-K（不要ウィジェット削除）完了後に着手する。

---

## L-1: ワークスペース編集用左サイドバー

### 背景

現在ウィジェットの追加は `AddWidgetDialog`（モーダル）のみ。フィードバックでは編集モード時に左サイドバーからウィジェットを D&D で追加できるようにすべきとされている。

### 受け入れ条件

- [x] 編集モードのオン/オフを切り替えるボタンが存在する
- [x] 編集モード時に左サイドバーが表示される
- [x] サイドバーに追加可能なウィジェットの一覧がアイコン＋名前で表示される
- [x] 既に配置済みのウィジェットはグレーアウト or 非表示（重複配置不可の場合）
- [x] 非編集モード時はサイドバーが非表示になる
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                              | 変更内容                              |
| --------------------------------------------------------------------- | ------------------------------------- |
| `src/lib/components/arcagate/workspace/WorkspaceEditorSidebar.svelte` | 新規作成 — ウィジェット一覧サイドバー |
| `src/lib/components/arcagate/workspace/WorkspaceLayout.svelte`        | 編集モード状態管理 + サイドバー表示   |

---

## L-2: D&D でウィジェット追加

### 背景

L-1 のサイドバーからウィジェットをワークスペースのグリッドにドラッグ＆ドロップで追加できるようにする。

### 受け入れ条件

- [x] サイドバーのウィジェットアイテムをドラッグしてグリッド上にドロップできる
- [x] ドロップ位置にウィジェットが追加される（DB に保存）
- [x] ドラッグ中のビジュアルフィードバック（ゴースト要素、ドロップ可能領域のハイライト）がある
- [x] `AddWidgetDialog` は編集モードでの追加手段として引き続き利用可能（フォールバック）
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                              | 変更内容               |
| --------------------------------------------------------------------- | ---------------------- |
| `src/lib/components/arcagate/workspace/WorkspaceEditorSidebar.svelte` | ドラッグソース実装     |
| `src/lib/components/workspace/WidgetGrid.svelte`                      | ドロップターゲット実装 |

---

## L-3: 編集中にグリッド線表示

### 背景

編集モード時にウィジェットの配置先を視覚的にわかりやすくするため、CSS グリッドオーバーレイを表示する。

### 受け入れ条件

- [x] 編集モード時にグリッド線（4カラム）が半透明で表示される
- [x] グリッド線はウィジェットの下に表示される（z-index 制御）
- [x] 非編集モード時はグリッド線が非表示になる
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                         | 変更内容                             |
| ------------------------------------------------ | ------------------------------------ |
| `src/lib/components/workspace/WidgetGrid.svelte` | 編集モード時の CSS grid overlay 追加 |

---

## L-4: ウィジェットのリサイズ UI

### 背景

バックエンドの `workspace_widgets` テーブルには `width` / `height` カラムが存在するが、フロントエンドにリサイズ操作の UI がない。`WidgetCard` に右下リサイズハンドルは実装済みだが、ドラッグによるリサイズ操作を完成させる。

### 受け入れ条件

- [x] 編集モード時にウィジェットの右下にリサイズハンドルが表示される
- [x] ハンドルをドラッグしてウィジェットのサイズ（列スパン × 行スパン）を変更できる
- [x] リサイズ中にプレビュー（影）が表示される
- [x] リサイズ結果が DB に保存される（`resizeWidget` IPC）
- [x] 最小サイズ（1×1）と最大サイズ（4×4）の制限がある
- [x] 非編集モード時はリサイズハンドルが非表示になる
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                         | 変更内容                           |
| ------------------------------------------------ | ---------------------------------- |
| `src/lib/components/workspace/WidgetCard.svelte` | リサイズハンドルのドラッグ操作実装 |
| `src/lib/components/workspace/WidgetGrid.svelte` | リサイズ中のプレビュー表示         |

---

## L-5: ウィジェットの移動 UI

### 背景

`@formkit/drag-and-drop` で並び替えは実装済みだが、グリッド上の任意位置への移動 UI（`position_x`, `position_y` の変更）が未完成。

### 受け入れ条件

- [x] 編集モード時にウィジェットをドラッグして任意のグリッド位置に移動できる
- [x] 移動中にドロップ先のプレビューが表示される
- [x] 移動結果が DB に保存される（`position_x`, `position_y` の更新）
- [x] 他のウィジェットとの重なり検出・自動再配置がある
- [x] 非編集モード時はドラッグ移動が無効になる
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                         | 変更内容                         |
| ------------------------------------------------ | -------------------------------- |
| `src/lib/components/workspace/WidgetGrid.svelte` | グリッド位置指定の移動実装       |
| `src/lib/components/workspace/WidgetCard.svelte` | ドラッグハンドルの編集モード制御 |
