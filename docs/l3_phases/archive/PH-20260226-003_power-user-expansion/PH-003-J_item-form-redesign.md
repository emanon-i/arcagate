# PH-003-J: アイテムフォーム刷新

## 概要

アイテム追加フォーム（`ItemForm.svelte` / `ItemFormDialog.svelte`）を大幅にリデザインする。
ドロップゾーンのクリック対応、URL/ローカル二択の自動推測、ターゲット readonly 化、
作業ディレクトリ削除、ファイル追跡チェックボックス追加、アイコン画像プレビュー、UI統一を行う。

---

## J-1: ドロップゾーンクリックでファイルダイアログ

### 背景

現在のドロップゾーン（`DropZone.svelte` / `QuickRegisterDropZone.svelte`）は D&D のみ対応。クリックでもファイル選択できるべき。

### 受け入れ条件

- [x] ドロップゾーンをクリックすると `tauri-plugin-dialog` のファイル選択ダイアログが開く
- [x] 選択されたファイルパスが D&D と同じフローで処理される（`handleDrop` と同一パス）
- [x] D&D による登録は引き続き動作する
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                                           | 変更内容                                                         |
| ------------------------------------------------------------------ | ---------------------------------------------------------------- |
| `src/lib/components/item/DropZone.svelte`                          | クリックハンドラ追加、`tauri-plugin-dialog` の `open()` 呼び出し |
| `src/lib/components/arcagate/library/QuickRegisterDropZone.svelte` | 同様のクリックハンドラ追加                                       |
| `src-tauri/Cargo.toml`                                             | `tauri-plugin-dialog` 依存追加（未導入の場合）                   |
| `src-tauri/capabilities/default.json`                              | dialog パーミッション追加（未設定の場合）                        |

---

## J-2: タイプは URL/ローカルの自動推測のみ

### 背景

現在は手動 select で 5 種類（exe, url, folder, script, command）を選択可能。フィードバックでは「タイプは自動推測のみ（URL vs ローカルの二択）」とされている。

### 受け入れ条件

- [x] タイプ選択 UI が URL / ローカル の二択トグルに変更される
- [x] ターゲット入力に基づいて自動でタイプが推測される（`detectType()` の活用）
- [x] 手動でのタイプ変更は URL ↔ ローカル の切替のみ可能
- [x] バックエンドの `ItemType` enum（exe, url, folder, script, command）はそのまま維持
- [x] フロントエンドの自動推測ロジック（`detect-type.ts`）がローカル内の細分化を担当
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                  | 変更内容                               |
| ----------------------------------------- | -------------------------------------- |
| `src/lib/components/item/ItemForm.svelte` | select → URL/ローカル トグル UI に変更 |
| `src/lib/utils/detect-type.ts`            | 必要に応じて推測ロジック調整           |

## J-3: ターゲットは自動入力・編集不可

### 背景

現在はターゲットフィールドが自由編集可能なテキスト入力。フィードバックでは「ターゲットは自動入力・編集不可」とされている。D&D またはファイルダイアログで設定し、直接編集は不可にする。

### 受け入れ条件

- [x] ターゲット入力フィールドが `readonly` になる
- [x] D&D またはファイルダイアログ（J-1）からのみ値が設定される
- [x] URL タイプの場合はテキスト入力を許可する（URL は D&D 不可のため）
- [x] `pnpm verify` が通過する

### 変更対象ファイル

| ファイル                                  | 変更内容                                                   |
| ----------------------------------------- | ---------------------------------------------------------- |
| `src/lib/components/item/ItemForm.svelte` | ターゲットフィールドに `readonly` 属性追加（URL 時は解除） |
