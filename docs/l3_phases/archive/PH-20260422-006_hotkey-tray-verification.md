# PH-20260422-006: グローバルホットキー + システムトレイ常駐・自動起動 実機検証

## 目的

REQ-20260226-003「システムトレイ常駐・グローバルホットキー・自動起動」は実装済みとされているが
E2E 経路で一度も通して検証されていない。毎日使えるソフトウェアとして必須の経路を実機で保証する。

## 検証スコープ

### 確認事項 A: グローバルホットキー

1. `pnpm tauri dev` でアプリ起動
2. アプリを**バックグラウンドに回す**（Alt+Tab で他ウィンドウへ）
3. 設定されたホットキー（デフォルト or SettingsPanel で設定したキー）を押下
4. コマンドパレットウィンドウが前面に出ることを確認
5. もう一度ホットキー押下で非表示になることを確認（トグル動作）

**期待動作**: どのウィンドウがアクティブでもホットキーでパレットが出る

### 確認事項 B: システムトレイアイコン常駐

1. アプリ起動後、タスクバーのシステムトレイにアイコンが表示されるか確認
2. メインウィンドウを × で閉じてもアプリがトレイに常駐し続けるか確認
3. トレイアイコン右クリックメニューの項目（表示・終了等）を確認

**期待動作**: × 閉じはトレイ常駐。完全終了は終了メニューから。

### 確認事項 C: 自動起動設定

1. SettingsPanel の自動起動トグルが存在するか確認
2. トグル ON にして設定を保存
3. アプリ終了 → Windows スタートアップに登録されているか確認
   （`shell:startup` フォルダ または タスクマネージャー > スタートアップタブ）

**期待動作**: トグル ON でスタートアップ登録される

## 修正方針

不具合が見つかった場合:

- ホットキーが動かない → `src-tauri/src/lib.rs` のホットキー登録コードを確認・修正
- トレイ常駐が動かない → Tauri tray 設定を確認
- 自動起動が動かない → `tauri-plugin-autostart` の設定を確認

いずれも **scope_files 内のみ** で解決する。プラグイン追加が必要な場合は停止して報告。
