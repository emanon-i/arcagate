## 背景/目的

`items.spec.ts` の `@smoke` テスト「IPC 経由でアイテムを作成すると一覧に反映されること」が
CI でフレーキーに失敗する。`page.reload()` + `waitForAppReady()` 後に
`library-card-{id}` が 10 秒以内に表示されないことがある。

`waitForAppReady` は setup-wizard 消滅のみを検知し、Library の非同期データロードを待たないのが根本原因。

## 実装内容

`tests/e2e/items.spec.ts` の `@smoke` テストで `page.reload()` 後に
Library 本体が表示されるまで明示的に待つ:

```typescript
await page.reload();
await page.waitForLoadState('domcontentloaded');
await waitForAppReady(page);
// Library データロード完了を待つ
await page.waitForSelector('[data-testid^="library-card-"]', { timeout: 15_000 });
// 作成したアイテムのカードを確認
await expect(page.getByTestId(`library-card-${item.id}`)).toBeVisible({ timeout: 5_000 });
```

または `timeout` を 10_000 → 20_000 に変更するシンプルな修正でも可。
より根本的な修正は `waitForSelector` を用いたアプローチ。
