# PH-448: SBOM 生成 (CycloneDX format)

## 問題

Codex Q4 #3: 配布時の SBOM (Software Bill of Materials) は供給網透明性 + 企業利用最低線。

## 改修

- `cargo cyclonedx` (cargo plugin) 導入
  - `cargo install cargo-cyclonedx` を CI に追加
- `pnpm npm-cyclonedx` で frontend 依存も SBOM 化
- scripts/generate-sbom.sh 新設:
  - cargo cyclonedx --output-format json --output-file sbom-rust.json
  - npx @cyclonedx/cyclonedx-npm --output-file sbom-npm.json
  - 統合: scripts/sbom-merge.js で 1 ファイル化
- Release workflow (PH-447) で SBOM も assets に添付
