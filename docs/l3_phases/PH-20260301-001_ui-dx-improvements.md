### F-20260301-001: フローティングコマンドパレット

ホットキー発火時にフルウィンドウではなく、画面中央に小さなフローティングオーバーレイとしてパレットだけを表示する。既存ソフト（PowerToys Run / Alfred）と同等の体験。

**技術要素**:

- Tauri の第2ウィンドウ（`palette` ウィンドウ）を `decorations: false`、`transparent: true`、`skip_taskbar: true` で作成
- メインウィンドウとは独立して表示/非表示をトグル
- ホットキー発火時に `palette` ウィンドウを `show()` + `set_focus()`、確定・Escで `hide()`
- ウィンドウ外クリックで閉じる（`on_window_event` の `Focused(false)` または JS の `blur` イベント）

**受け入れ条件**:

- [ ] ホットキー発火時にメインウィンドウとは別のフローティングウィンドウとしてパレットが表示される
- [ ] パレットウィンドウはタスクバー・Altタブに表示されない
- [ ] 検索→起動後、またはEscキーでパレットウィンドウが閉じる
- [ ] ウィンドウ外をクリックするとパレットが閉じる
- [ ] メインウィンドウはホットキーに影響されず独立して動作する

### F-20260301-002: アイテム一覧画面からのD&D登録

アイテム一覧画面に直接ファイルをドロップすると、値が事前入力された状態で登録ダイアログが開く。フォームを開いてからドロップする手順を省略する。

**技術要素**:

- `+page.svelte` のアイテムタブ領域に `tauri://drag-drop` リスナーを追加
- ドロップ検出 → `editingItem = null` + フォームの初期値セット + `showItemForm = true`
- 既存の `DropZone.svelte` をフォーム内からページレベルに移動（または両方対応）

**受け入れ条件**:

- [ ] アイテム一覧画面にファイルをドロップするとダイアログが開く
- [ ] ダイアログ表示時点でターゲット・ラベル・タイプが自動入力されている
- [ ] ドロップ中はアイテム一覧エリアにドロップ可能であることを示すハイライトが表示される

### F-20260301-003: テーマシステム

ダーク/ライトモードの切り替えと、カラーテーマのカスタマイズ・インポートをサポートする。

**技術要素**:

- Tailwind CSS の `dark:` クラス + `document.documentElement.classList` で ダーク/ライト切替
- テーマ定義: CSS カスタムプロパティ（`--primary`, `--background` 等）を JSON で管理
- テーマ設定を `config` テーブルに保存（`key: "theme"`, `value: JSON文字列`）
- テーマインポート: JSON ファイルを `tauri-plugin-dialog` で選択し読み込む
- 設定画面にテーマセクションを追加

**受け入れ条件**:

- [ ] 設定画面でダーク/ライト/システム連動の3モードを切り替えられる
- [ ] 切り替えはリアルタイムに反映され、再起動後も保持される
- [ ] プリセットテーマ（最低2種）を選択できる
- [ ] テーマ定義JSONをファイルからインポートして適用できる

### F-20260301-004: ファイル拡張子による自動タイプ判定

アイテム登録フォームのターゲット欄に手入力・貼り付けしたときも、ファイル拡張子からタイプを自動判定してセレクトボックスを更新する。

**技術要素**:

- `target` の `$effect` またはイベントハンドラで拡張子を解析
- 判定ルール: `.exe` → exe、`.ps1/.bat/.cmd` → script、`http(s)://` → url、パス末尾が区切り文字またはドライブルート → folder、その他 → command
- D&D の `detectType()` と共通化

**受け入れ条件**:

- [ ] ターゲット欄に `.exe` パスを入力するとタイプが `exe` に自動更新される
- [ ] `https://` から始まる文字列を入力するとタイプが `url` に自動更新される
- [ ] `.ps1`/`.bat`/`.cmd` のパスを入力するとタイプが `script` に自動更新される
- [ ] ユーザーが手動でタイプを変更した後は自動判定が上書きしない

### F-20260301-005: カスタムウィンドウフレーム

OS 標準のタイトルバーをカスタム実装に置き換え、アプリのデザインに統合された外観にする。

**技術要素**:

- `tauri.conf.json` の `decorations: false` でネイティブフレームを無効化
- Svelte でカスタムタイトルバーコンポーネント（`TitleBar.svelte`）を実装
  - ドラッグ移動: `data-tauri-drag-region` 属性
  - 最小化・最大化・閉じるボタン: `window.__TAURI__.window.getCurrent()` の `minimize()`/`maximize()`/`hide()`
- ウィンドウリサイズは OS 標準のリサイズハンドルを活用（または `resizable: true` のまま）

**受け入れ条件**:

- [ ] OS 標準タイトルバーが非表示になり、カスタムタイトルバーが表示される
- [ ] タイトルバーのドラッグでウィンドウを移動できる
- [ ] 最小化・閉じるボタンが機能する（閉じるはトレイに格納）
- [ ] テーマに合わせてタイトルバーの色が変わる

### F-20260301-006: アイコン表示

アイテム一覧とコマンドパレットの各行にアイコンを表示する。`icon_path` が設定済みの場合はその画像を、未設定の場合はタイプに応じたフォールバックアイコンを表示する。

**技術要素**:

- Tauri の `convertFileSrc()` で ローカルパスを WebView で表示可能な URL に変換
- フォールバック: タイプ別の SVG アイコン（exe / url / folder / script / command）
- `ItemList.svelte` と `ResultItem.svelte` に `<img>` または `<svg>` を追加

**受け入れ条件**:

- [ ] アイテム一覧の各行にアイコンが表示される
- [ ] コマンドパレットの各結果行にアイコンが表示される
- [ ] `icon_path` 未設定のアイテムにはタイプ別フォールバックアイコンが表示される
- [ ] 存在しないパスが設定されている場合もフォールバックアイコンが表示されクラッシュしない

### F-20260301-007: フィードバック改善

削除・起動など操作の結果をユーザーに伝えるフィードバックを整備する。

**技術要素**:

- トースト通知コンポーネント（`Toast.svelte`）: 画面右下に数秒表示してフェードアウト
- `toastStore`（`$state` ベース）: success / error / info の3種
- 削除時: 確認ダイアログ（既存の `tauri-plugin-dialog` の `ask()` を利用）
- 起動成功/失敗時に `launchItem` の結果に応じてトーストを表示

**受け入れ条件**:

- [ ] アイテム削除時に「本当に削除しますか？」確認ダイアログが表示される
- [ ] アイテム起動成功時に「[ラベル] を起動しました」トーストが表示される
- [ ] アイテム起動失敗時にエラー内容のトーストが表示される
- [ ] トーストは3秒後に自動で消える

### F-20260301-008: 操作性改善

キーボード操作・一覧表示の細かな使い勝手を改善する。

**技術要素**:

- `Esc` キーで開いているダイアログ・パレットを閉じる（`keydown` リスナー）
- パレット空検索時に最近使った/頻度の高いアイテムを候補として表示（`cmd_list_recent` / `cmd_list_frequent` を利用）
- アイテム一覧の `is_enabled` 列にインラインのトグルスイッチを追加（`cmd_update_item` で更新）
- アイテム一覧にタイプ・カテゴリのフィルタドロップダウンを追加（フロントエンドのみでフィルタ）

**受け入れ条件**:

- [ ] ダイアログ表示中に `Esc` キーを押すと閉じる
- [ ] パレットを開いて何も入力しない状態で最近/頻繁に使ったアイテムが表示される
- [ ] アイテム一覧の有効/無効をトグルスイッチで即時切り替えできる
- [ ] アイテム一覧をタイプ・カテゴリでフィルタできる
