# Arcagate UX デザインビジョン

作成: 2026-04-23 / 調査バッチ batch-31 (PH-143)

---

## 1. 背景と目標

Arcagate はデスクトップに散在した起動元を一箇所に集約する**個人用コントロールパネル**である。
機能面（ランチャー・ワークスペース・コマンドパレット）が充実してきた現時点で、
次のフェーズは **「触り心地」「気持ちよさ」「洗練されたわかりやすさ」** の追求である。

### 目標メトリクス（主観指標）

| 指標                 | 現状                   | 目標                                           |
| -------------------- | ---------------------- | ---------------------------------------------- |
| 開いたときの第一印象 | 機能的だが質素         | 「ちゃんとしたツール」という確信を即座に与える |
| 編集時の気持ちよさ   | 操作は通るが反応が薄い | D&D やリサイズが手に吸い付く感覚               |
| 長時間使用時の疲労度 | 問題なし               | 意図的に落ち着いた色・動きで疲労を出さない     |
| テーマ切替の楽しさ   | ダーク/ライトのみ      | プリセットで「雰囲気が変わる」体験             |

## 2. ゲーム UI/UX から採れる設計原則

ゲーム UI は「プレイヤーにとって快適な操作感」を極限まで追求した分野であり、
デスクトップツールに転用できる原則が多い。

### 2-1. Juice（反応の豊かさ）

> 「どんな操作も、何かが起きたことをプレイヤーに伝える」

- ボタンクリック → わずかなスケール変化
- リスト項目ホバー → 背景のなめらかなフェード
- ドロップ成功 → 軽い弾む動き

Arcagate への適用: **マイクロインタラクション**をすべての CTA に付与する。
`transition-all duration-150` + `hover:scale-[1.02]` レベルの小さな変化で十分。

### 2-2. フィードバックループ

ゲームでは「操作 → 状態変化 → 視覚/音声フィードバック → 次の操作」の
サイクルが必ず完結する。

デスクトップツールで壊れやすい箇所:

- ドラッグ中の「今どこに置けるか」の不明瞭さ
- 保存・削除後の「完了した」感の欠如

Arcagate への適用:

- ドラッグオーバー時のグロー/ハイライトを強化
- 破壊的操作（削除確認後）にトーストアニメーションを追加

### 2-3. モーションデザイン

ゲーム UI のモーション三原則:

1. **Purposeful（目的がある）**: 状態遷移を伝えるためだけに使う
2. **Responsive（即応）**: 入力から 100ms 以内に何かが動き始める
3. **Consistent（一貫）**: 同種の操作には同じ easing / duration

デスクトップツールへの転用では「速さ優先」。ゲームより短い duration で。

| 操作種別               | 推奨 duration | easing                                              |
| ---------------------- | ------------- | --------------------------------------------------- |
| ホバー・フォーカス     | 120〜150ms    | cubic-bezier(0.25, 0.46, 0.45, 0.94)                |
| パネル出現/消去        | 200〜250ms    | cubic-bezier(0.25, 0.46, 0.45, 0.94)                |
| ダイアログ出現         | 200ms         | cubic-bezier(0.25, 0.46, 0.45, 0.94) + scale 0.96→1 |
| ドラッグフィードバック | 80ms          | linear                                              |
| ページ遷移             | 200ms         | cubic-bezier(0.25, 0.46, 0.45, 0.94)                |

easing は「動き始めと終わりが緩やかに入る」**中割スタイル**を統一採用。
日本のリミテッドアニメーション（ジブリ・京アニ）の中割タイミングに近い、
両端が遅くなだらかにつながる曲線。ドラッグのみ即応を優先して `linear`。

### 2-4. サウンドデザイン

ゲームでは常時・毎操作に音が鳴る。Arcagate では「**鳴らすべき音だけ鳴らす**」方針。

採用可否の原則:

- **ホバー・連続入力は NG**（連続発火で疲労感、OS 効果音と競合）
- **プライマリクリック SE は採用**（ボタン確定・パレット実行）→ ゲーム的な触感フィードバックとして有効
- **重要イベントも採用**（起動成功・削除確定）→ 操作の完了を耳で確認できる
- プライマリクリック SE は**デフォルト ON** を推奨。うるさければ設定でまとめて OFF 可。
- 音量は 0.0〜1.0 スライダー（設定画面）で制御。

### 2-5. 情報密度とコントラスト

ゲーム UI は高密度でも可読性を保つために:

- **階層的な視線誘導**: 最重要 → 中程度 → 補足の 3 段階
- **コントラスト比**: WCAG AA（4.5:1）以上を常に維持
- **視線が流れる方向**: 左→右、上→下を基本とし、重要な UI は左上に

Arcagate への適用:

- `--ag-text-primary` / `secondary` / `muted` / `faint` の 4 段階は維持
- アクセントカラーは「本当に注意を引きたい場所だけ」に制限

## 3. Arknights Endfield の視覚言語分析

### 3-1. 概観

- ジャンル: 3D ストラテジー RPG / インフラ建設
- UI テイスト: ハードサイファイ（SF 軍事系）+ 有機的な自然テクスチャの混在

### 3-2. 背景・テクスチャ表現

- **コンタライン（等高線）**: 地形の等高線を抽象化した線が UI 背景にオーバーレイされる
  - 薄い半透明ライン、opacity 5〜15%
  - 静止状態では動かず、パネル出現時に微かに流れる
- **ノイズテクスチャ**: 単色塗りのパネルにフィルムグレインや砂目状ノイズを重ねる
  - Gaussian ノイズ opacity 3〜8%
  - SVG filter の `feTurbulence` または CSS backdrop-filter で実現可能
- **境界線の光**: パネルの上端・左端に 1px の明るい線（ライトストローク）を入れる
  - ガラス的な質感の演出

### 3-3. カラーパレット

| 役割       | Endfield 系           | Arcagate 現行                    |
| ---------- | --------------------- | -------------------------------- |
| 背景ベース | 深い青灰 (#0d1117 系) | #090b10 (dark) / #f8f9fb (light) |
| サーフェス | 暗めの青灰レイヤー    | rgba(255,255,255,0.03〜0.05)     |
| アクセント | 青緑系シアン          | cyan (#22d3ee)                   |
| 警告/注意  | 琥珀系オレンジ        | #fbbf24 系                       |
| テキスト   | 冷たい白 (blue-tint)  | rgba(255,255,255,0.95)           |

→ 現行の `--ag-*` トークンは Endfield テイストとすでに近い。

### 3-4. タイポグラフィ

- 欧文: モノスペース + セリフなしサンセリフの混用
- 日本語: システムフォントで十分（カスタムフォントはパフォーマンス懸念あり）
- 特徴的なラベル: ALL CAPS + 0.1em 程度の letter-spacing（ `text-xs tracking-widest` ）

### 3-5. アニメーション特性

- パネルが出現する際: わずかに上からスライドイン (translateY -8px → 0) + フェード
- 選択状態: 背景に低opacity のアクセントカラー + 左ボーダーライン 2px
- ホバー状態: 背景 opacity を 1 段階上げる（surface-2 → surface-3 相当）
