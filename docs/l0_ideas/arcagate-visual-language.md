1. **Arknights Endfield** — 等高線ベース背景 + テクスチャ / ノイズ感
   - 暗く落ち着いたトーン（深い青灰）

   - ハードウェア感・機械感がほんのりある質感

   - クリーンな角丸ウィンドウ + スクロールバーが目立たない

## Arknights Endfield 視覚言語 詳細記録

### カラーパレット分析

```
ベース背景色
  深い青灰: #0a0e14 〜 #0d1117
  中間サーフェス: #141a24 〜 #1a2232

テキスト
  プライマリ: rgba(240,248,255, 0.95)  ← わずかに青みがかった白
  セカンダリ: rgba(180,200,220, 0.65)
  ミュート: rgba(120,150,180, 0.45)

アクセント（cyan 系）
  ハイライト: #22d3ee / #67e8f9
  ボーダー: rgba(34,211,238, 0.25)

サブアクセント（amber 系: 警告・重要）
  テキスト: #fbbf24
  ボーダー: rgba(251,191,36, 0.3)
```

→ 現行の Arcagate dark テーマはこの方向性とほぼ一致している。
追加変更は「青みの強化」「ノイズテクスチャ追加」程度で達成可能。

### 背景テクスチャ再現方針

```
手法 A: SVG feTurbulence フィルタ
<svg>
  <filter id="noise">
    <feTurbulence type="fractalNoise" baseFrequency="0.65" numOctaves="3" stitchTiles="stitch"/>
    <feColorMatrix type="saturate" values="0"/>
    <feBlend in="SourceGraphic" mode="overlay"/>
  </filter>
</svg>

手法 B: CSS background-image (PNG スプライト)
background-image: url("noise.png");
background-size: 200px 200px;
opacity: 0.05;

手法 C: data-URL インライン（外部ファイル不要）
```

→ 手法 A が最も軽量でスケーラブル。SVG は `include_str!` でバイナリに埋め込み可能。

### 等高線（コンタライン）再現方針

```css
/* CSS のみで実現する等高線的パターン */
background-image: repeating-linear-gradient(
  0deg,
  transparent,
  transparent 19px,
  rgba(255,255,255,0.04) 19px,
  rgba(255,255,255,0.04) 20px
);
```

→ 本物の等高線は SVG + JS アニメーションが必要。コスト高なので初期は CSS 疑似表現で。

## Ubuntu 系ビジュアル 詳細記録

### Frosted Glass（Arcagate 採用候補）

```css
/* パネルに適用する Frosted Glass スタイル */
.frosted-panel {
  background: rgba(10, 14, 20, 0.7);    /* 70% 不透明のダーク */
  backdrop-filter: blur(12px) saturate(180%);
  -webkit-backdrop-filter: blur(12px) saturate(180%);
  border: 1px solid rgba(255, 255, 255, 0.08);
}
```

WebView2 は `backdrop-filter` に対応しているため実装可能。
ただし**スクロールエリアでのパフォーマンス低下**に注意（スクロール中は blur 無効化の workaround を検討）。

### 採用しないエフェクトと理由

| エフェクト            | 不採用理由                                |
| --------------------- | ----------------------------------------- |
| 雨粒エフェクト        | 常時 GPU 使用・疲労感・ツールとして不適切 |
| パーティクル          | 同上                                      |
| スキャンライン        | ゲームっぽすぎる・読みにくい              |
| CRT エフェクト        | 同上                                      |
| 不透明度 < 0.5 の本文 | WCAG 違反・可読性不足                     |

## Arcagate ブランドの方向性

### 目指すキャラクター

> 「よく磨かれた工具」

機械的で冷たいのではなく、**手に馴染む精密工具**のような印象。
使うほどに信頼できる、飽きのこないデザイン。

### 避けるキャラクター

- 過度に「ゲームっぽい」（プレイヤーではなく制作者向けツール）
- 過度に「ミニマル」（機能が見えない、何もできなさそう）
- 過度に「派手」（毎日使うには疲れる）

### カラーの方向性

```
ダークテーマ: 深い青灰ベース × シアンアクセント（現行維持・強化）
ライトテーマ: 冷たいグレーホワイト × シアンアクセント（現行維持）
Endfield プリセット: より深い青灰 + ノイズテクスチャ追加
Frosted プリセット: 透過感 + 壁紙ブラー（将来）
```

## セッション間での参照方法

このドキュメントは `docs/l0_ideas/arcagate-visual-language.md` に永続保管。
デザインの方向性で迷ったら必ずここを確認する。

**重要な制約（MUST）:**

- コンポーネントに色コードをハードコードしない（必ず `--ag-*` を経由）
- 可読性を損なうデザインは採用しない
- パフォーマンス（idle 100MB / 起動 2秒）を優先
- デフォルトはシンプルで、エフェクトは設定で有効化できる形にする
