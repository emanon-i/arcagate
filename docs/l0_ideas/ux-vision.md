# Arcagate UX ビジョン

## デザインキャラクター

「よく磨かれた工具」。精密で信頼できるが、冷たくはない。毎日使うほど手に馴染む。

**避けたいもの:**

- 過度にゲームっぽい（制作者向けツールなのでプレイヤー向けにしない）
- 過度にミニマル（何もできなさそうに見える）
- 過度に派手（毎日使うには疲れる）

## 何を大事にするか

操作が確実に動くこと、操作した結果が見えること、この 2 点が最優先。見た目の洗練は操作性・理解可能性を満たした後で意味を持つ。

## アニメーションの方針

「動き始めと終わりが緩やかにつながる」中割スタイルを基本とする。日本のリミテッドアニメーション（ジブリ・京アニ）の easing に近い曲線。ドラッグだけは即応性優先で linear。

早さ優先。ゲームより短い duration で。100ms 以内に視覚的反応が始まること。

## 視覚エフェクトの採用/却下基準

**採用:**

- マイクロインタラクション（ホバー・クリックのわずかな変化）— 全 CTA に
- ダイアログ/パネルの出現アニメーション（scale + fade）
- D&D ドロップゾーンのハイライト
- トーストのスライドアニメーション
- Frosted Glass（設定で切り替え可能）
- 背景ノイズテクスチャ（ダーク時の微細グレイン、オプション）

**却下（理由付き）:**

- 常時パーティクル・雨粒: 疲労感、ツール用途に不適
- スキャンライン・CRT エフェクト: ゲームすぎる、可読性低下
- 常時流れるアニメーション（コンタライン等）: CPU 負荷、長時間使用で疲労
- BGM・環境音: デスクトップツールに不要

**検討中（実装時に判断）:**

- シマー（ローディング時の光走り）: 頻出すると疲労感あり
- リップル（クリック波紋）: ゲームっぽくなりすぎる可能性

## サウンドの方針

現状は未実装。将来的に採用する場合:

- プライマリクリック SE とパレット実行音はデフォルト ON（ゲーム的な触感として楽しい）
- ホバー・連続入力への音は NG（連続発火で疲労感）
- 全て Web Audio API（外部ファイル不依存）
- 設定でまとめて OFF 可能

## Reduced Motion 対応

`prefers-reduced-motion: reduce` 時は全アニメーション duration を 0ms にする。すべてのアニメーション要素に `motion-reduce:transition-none` を付与する。
