# Arcagate（アルカゲート）— Concept Document

---

## 1. 概要

PC上に散在する起動元（Steam、各種ランチャー、ブラウザ、スクリプト）を一箇所に集め、登録・整理・起動できる個人用コントロールパネル。コマンドパレット中心の軽量常駐アプリとして始まり、ワークスペース・CLI連携へと段階的に育てる。

---

## 2. 課題と目的

### 課題

- アプリの起動元がバラバラ（Steam、DMMゲーム、ブラウザゲーム、各種ランチャー等）で一元管理できていない
- 同一アプリの複数バージョン（Blender 3.x / 4.x 等）の切り替えが煩雑
- PowerShellスクリプトやClaude Code起動などの定型操作に手数がかかる
- 既存のランチャー代替ツールは、モダンなUIでないか動作が重いか、開発ツールとゲームを同列に扱えない

### 目的

「起動元・種別を問わず、何でも登録して即座に起動できる」個人用ハブを構築する。毎日開くアプリとして常駐させ、起動の手間を削減する。起動ログは回数・最終日時の最小限のみ自前で持つ。**品質バーは配布・販売に耐える水準**を狙い、現状は開発者自身の毎日の使用を起点に磨き込む。GitHub public で公開状態であり、将来的な配布・販売はオプションとして開かれている。

---

## 3. ユースケース

### M1で解決されるシナリオ

- **ゲーム起動の一元化**：普段はSteam/DMMと複数のランチャーを行き来しているが、Arcagateだけを開けば全ゲームにアクセスできる
- **Blender複数バージョン切り替え**：`blen3` / `blen4` と打ち分けるだけで目的のバージョンが起動する
- **Claude Codeのワンクリック起動**：PowerShellスクリプトをArcagateに登録し、コマンドパレットから即実行できる
- **ホットキー起動フロー**：どの画面にいてもホットキー一発でパレットが開き、名前を打ってEnterで起動する（これがM1のキラー体験）

### M2以降で実現するシナリオ

- **ツール専用ワークスペース**：「Obsidianワークスペース」ページにボルト別ショートカット・関連スクリプトを集約して管理する
- **Claude CodeからCLI経由で操作**：「Blender起動して」「開発用ワークスペース開いて」を自然言語で指示し、Agent-first CLI（arcagate_cli）が実行する
- **~~コンテキストメニューから自作スクリプト呼び出し（M3）~~**：⚠️ 凍結中（PH-004 で凍結。優先度見直し待ち）
- **監視フォルダによる自動登録（M2b）**：監視フォルダに追加されたアイテムをランチャーに自動登録（M2b で実現済み）

## 4. 前提条件・制約

| 項目             | 内容                                                                  |
| ---------------- | --------------------------------------------------------------------- |
| コスト           | ゼロコスト運用（OSS・無料サービスのみ）                               |
| プラットフォーム | Windows x86-64 優先、将来的なクロスプラットフォームを視野に入れた設計 |
| 技術スタック     | Tauri + Svelte（UI）+ Rust（バックエンド）+ SQLite（ローカルDB）      |
| 常駐メモリ       | Idle時 Working Set 100MB以下（M1目標）                                |
| 起動時間         | ホットキー押下からUI表示まで P95 2秒以内（M1目標）                    |
| バイナリサイズ   | インストーラーなし単体exe 20MB以下                                    |
| スコープ管理     | M1を最優先とし段階的に開発。M1完成前にM2に着手しない                  |

※ 配布・販売前提になる場合は code signing 等が別途必要。

## 5. コアコンセプト

### 「登録できれば起動できる」—— 種別を問わない統一アイテムモデル

既存のランチャーは「ゲームランチャー」「開発ツール管理」「スクリプトランナー」が別アプリとして存在し、横断管理ができない。Arcagateは `.exe / URL / フォルダ / スクリプト / コマンド` を同一のアイテムモデルで扱い、「登録できれば何でも起動できる」を原則とする。これにより、Steam・ブラウザゲーム・PowerShellスクリプト・Claude Codeを単一の場所から扱える。

### コマンドパレット中心設計

UIの核はコマンドパレット（キーボードインターフェース）。マウスでも使えるが、「ホットキー → 名前を打つ → Enter」のフローが最速経路になるよう設計する。ワークスペース（M2b以降）はコマンドパレットを補完する位置付けであり、逆転させない。

### プラグインアーキテクチャ——コアは最小、機能は追加可能

コアが直接持つのは「アイテムの登録・起動・常駐」のみ。コマンドパレット拡張・ワークスペース・CLIはすべてプラグインとして実装する。公式プラグインとサードパーティプラグインは同一のインターフェースで動作する。M1はプラグイン構造なしで実装するが、インターフェース境界の定義だけはM1設計時に行い、M2移行時の全面リファクタを防ぐ。

### サービスレイヤーの一元化

UI・CLIは同一のサービス層を経由する。これにより、UI操作でもCLIでも同じ操作結果になることを保証する。

## 6. 機能要件

マイルストーンごとに定義する。詳細はセクション8（マイルストーン）と合わせて参照。

### M1：登録・起動・常駐

- 任意の実行ファイル・URL・フォルダ・スクリプトを登録できる
- ラベル・アイコン・カテゴリ・検索用別名を設定できる
- exeのドラッグ&ドロップで即登録できる（アイコン自動取得付き）
- コマンドパレットから名前で検索・起動できる
- システムトレイに常駐し、ホットキーでどこからでも呼び出せる
- Windows起動時に自動起動できる
- 起動回数・最終起動日時をSQLiteに記録し、頻度・最近使ったもの順に表示できる
- SQLiteデータ一式をエクスポート/インポートできる
- 初回起動時のシンプルなセットアップウィザードがある
- タグに「デフォルト非表示」フラグを設定でき、ホットキーまたはパスワードでトグル表示できる（センシティブなコンテンツの隠蔽用途）

### M2a：コマンドパレット拡張

- CLIからコマンドパレットを操作できる（例：`arcagate run blender`）
- クリップボード履歴・スニペット・電卓を内蔵コマンドとして使える
- カテゴリにプレフィックスを設定し名前空間として使える（例：`gm` → Gamesカテゴリの絞り込み）
- ファイルシステム監視によるパス変更の自動追跡ができる

### M2b：ワークスペース

- ユーザーが任意のページを作成し、ウィジェットを自由に配置できる
- よく使うもの・最近使ったもの・プロジェクト一覧・監視フォルダを内蔵ウィジェットとして使える
- テーマ（ダーク・ライト・カスタムカラー）を切り替えられる
- UIの角丸・ボタンの丸み等のシェイプをテーマごとに設定できる
- テーマをインポート・エクスポートして共有できる
- 監視フォルダに追加されたアイテムをランチャーに自動登録できる
- Gitステータス（ブランチ名・未コミット変更・最終コミット日時）をプロジェクト一覧で確認できる

### M2c：Agent-first CLI

> PH-003-H で MCP サーバーを除去し、Agent-first CLI に移行済み。

- `arcagate_cli` が Claude Code の Agent tool / Skill から直接呼び出される
- Claude Code から `arcagate_cli run <name>` / `arcagate_cli search <query>` 等で操作可能
- MCP プロトコルの複雑さを排除し、シンプルな CLI インターフェースで同等の機能を実現

## 7. スコープ外

- クラウド同期（ローカル完結が原則）
- Linux / macOS ネイティブ対応（設計上は意識するが対応しない）
- Steamライブラリ等との自動同期（手動登録が基本だが、将来対応の可能性あり）
- ターミナルエミュレータ統合（PTY・エンコーディング作り込みは対象外。ログを別ウィンドウで開く程度に留める）
- Google Calendar・タスクリスト連携（将来プラグインとして対応可能だが初期は対象外）
