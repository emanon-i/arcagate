# i18n

> cross-cutting / 規範本体は [`../../i18n-policy.md`](../../i18n-policy.md) (i18n Policy)

## 目的

hardcoded string を「英語 keep / 翻訳 / icon 化 / dedup」のいずれにするか、「2 人が適用しても同じ答え」になる mechanical な判定で決める規範。Microsoft Store + 海外展開対応。

## やること (必要処理)

- 文字列を decision tree (6 step) で分類: (A) literal keep / (B) ja===en / (C) 翻訳 / (D) icon-only
- 3 基準で判定: 配置場所 (UI slot で言語固定) → 文法構造 (述語 / 助詞 → 日本語必須) → 文字数 (英語 3 単語 / 25 文字以内)
- (C) は `t('area.sub.name')` で `messages_ja.json` / `messages_en.json` に key 化
- 2 箇所以上の同一文言は `common.*` に dedup

## やらないこと (禁止 / scope 外)

- 「なんとなく英語のまま」= 3 基準で説明できない英語 hardcode をしない
- brand / 技術略語 / hotkey / 拡張子 / version を t() 化しない (過剰翻訳)
- 同一 JP 文言を area-specific key で重複作成しない (dedup 漏れ)
- universal icon 該当を text-only button にしない (icon 化漏れ)
- rule なしの無意識な言語混在をしない (rule-based 混在は OK)

## 性能予算

- `t(key)` は in-memory message lookup。起動時に locale 1 回 detect

## 副作用 (state 変化 / persistence)

- locale は OS auto-detect + 設定上書き
- ⚠️ e2e test では locale を強制する (OS auto-detect + ja-hardcode selector の組合せが破壊的 — feedback memory)

## 依存

- 実装: `src/lib/i18n.svelte.ts` / `src/lib/i18n/messages_{ja,en}.json`
- audit: `scripts/audit-i18n-hardcode.sh` (pre-commit、hardcoded JP budget gate)
- 整合: `../../button-usage.md` (icon-only 化判定)

## 既知の判断

- widget registry 名 / form label は (C) 日本語、nav menu / empty-state title は (B) ja===en
- 詳細な decision tree・判定例は規範本体 [`../../i18n-policy.md`](../../i18n-policy.md) を参照
